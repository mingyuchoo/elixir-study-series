## T055: 캐러셀 자동 진행 테스트

### 테스트 URL
<http://localhost:4000>

### 테스트 절차

1. **자동 진행 확인**
   - 웹 브라우저에서 <http://localhost:4000> 접속
   - 홈페이지 상단의 캐러셀 확인 (어두운 보라-파랑 배경)
   - 아무 조작 없이 기다리기
   - ✅ 5초마다 자동으로 슬라이드가 변경되는지 확인

2. **일시정지 기능 확인**
   - 마우스를 캐러셀 위에 올리기
   - ✅ 자동 진행이 멈추는지 확인
   - 마우스를 캐러셀에서 떼기
   - ✅ 자동 진행이 재개되는지 확인

3. **수동 네비게이션 확인**
   - 좌우 네비게이션 버튼(화살표) 클릭
   - ✅ 슬라이드가 수동으로 변경되는지 확인
   - 하단 인디케이터 도트 클릭
   - ✅ 특정 슬라이드로 이동하는지 확인

### 예상 결과

- 캐러셀이 5초마다 자동으로 전환됨
- 부드러운 opacity 전환 효과
- 마우스 hover 시 일시정지, leave 시 재개
- 네비게이션 버튼과 인디케이터가 정상 작동

## T056: 이메일 구독 메시지 테스트

### 테스트 URL
<http://localhost:4000> (페이지 하단으로 스크롤)

### 테스트 1: 성공 메시지

1. 페이지 하단의 보라색 구독 섹션으로 스크롤
2. "최신 글을 이메일로 받아보세요" 제목 확인
3. 이메일 입력 필드에 새로운 이메일 주소 입력

   ```
   예: test-success@example.com
   ```

4. "구독하기" 버튼 클릭
5. ✅ **확인**: "✓ 구독이 완료되었습니다! 감사합니다." 메시지가 Elixir로 표시됨

### 테스트 2: 중복 이메일 메시지

1. 페이지 새로고침 (F5 또는 Ctrl+R)
2. 페이지 하단으로 다시 스크롤
3. 이메일 입력 필드에 **같은 이메일 주소** 입력

   ```
   예: test-success@example.com
   ```

4. "구독하기" 버튼 클릭
5. ✅ **확인**: "이미 구독하신 이메일입니다." 메시지가 Elixir로 표시됨

### 테스트 3: 잘못된 이메일 형식

1. 이메일 입력 필드에 잘못된 형식 입력

   ```
   예: invalid-email-format
   ```

2. "구독하기" 버튼 클릭
3. ✅ **확인**: 브라우저 또는 서버 검증 오류 메시지 표시

### 예상 결과

- 성공 메시지: 흰색 텍스트, 반투명 배경
- 중복 메시지: 흰색 텍스트, 반투명 배경
- 오류 메시지: 흰색 텍스트, 빨간색 배경
- 모든 메시지가 LiveView로 실시간 업데이트됨
