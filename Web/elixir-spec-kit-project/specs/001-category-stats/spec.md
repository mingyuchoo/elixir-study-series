# Feature Specification: 카테고리별 포스트 통계 페이지

**Feature Branch**: `001-category-stats`
**Created**: 2025-12-30
**Status**: Draft
**Input**: User description: "블로그 우측 상단에 있는 '카테고리' 를 누르면 저장되어 있는 태그별로 등록된 포스트 통계가 나와야 한다."

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 카테고리 통계 개요 페이지 조회 (Priority: P1)

독자가 블로그 상단의 '카테고리' 링크를 클릭하면 전체 카테고리(태그)별 포스트 통계를 한눈에 볼 수 있다. 이 페이지는 인기 포스트 통계와 카테고리별 포스트 수를 그리드 형태로 보여준다.

**Why this priority**: 사용자가 블로그의 콘텐츠 구조를 이해하고 관심 주제를 빠르게 찾을 수 있는 핵심 기능이다. 카테고리 페이지 없이는 사용자가 특정 주제의 글을 체계적으로 탐색할 수 없다.

**Independent Test**: 카테고리 메뉴를 클릭하여 통계 페이지가 로드되고, 각 카테고리의 포스트 수가 정확하게 표시되는지 확인할 수 있다.

**Acceptance Scenarios**:

1. **Given** 블로그 홈페이지에 접속한 상태, **When** 우측 상단의 '카테고리' 링크를 클릭, **Then** 카테고리 통계 페이지로 이동하여 인기 블로그 통계 Grid와 카테고리별 블로그 통계 Grid가 표시된다
2. **Given** 카테고리 통계 페이지에 접속한 상태, **When** 페이지가 로드됨, **Then** 각 카테고리 항목에 태그 이름과 해당 태그의 포스트 수가 표시된다
3. **Given** 여러 태그가 존재하는 상태, **When** 카테고리 페이지를 조회, **Then** 모든 태그가 알파벳 또는 가나다 순으로 정렬되어 표시된다
4. **Given** 특정 태그에 포스트가 없는 상태, **When** 카테고리 페이지를 조회, **Then** 해당 태그도 포스트 수 0으로 표시된다

---

### User Story 2 - 인기 포스트 통계 표시 (Priority: P2)

카테고리 통계 페이지 상단에 인기 포스트 통계 섹션이 표시되어 독자가 현재 인기 있는 콘텐츠를 빠르게 확인할 수 있다.

**Why this priority**: 인기 콘텐츠는 독자에게 양질의 글을 추천하고 사이트 체류 시간을 늘리는 데 도움이 되지만, 카테고리별 통계가 더 핵심적인 기능이므로 P2 우선순위를 부여한다.

**Independent Test**: 카테고리 페이지에서 인기 포스트 섹션이 독립적으로 표시되며, is_popular 플래그가 설정된 포스트들만 나타나는지 확인할 수 있다.

**Acceptance Scenarios**:

1. **Given** is_popular 플래그가 true인 포스트들이 존재하는 상태, **When** 카테고리 페이지를 조회, **Then** 페이지 상단에 인기 블로그 통계 Grid가 표시된다
2. **Given** 인기 포스트가 여러 개 존재하는 상태, **When** 인기 블로그 통계 섹션을 조회, **Then** 포스트들이 발행일 기준 최신순으로 정렬되어 표시된다
3. **Given** 인기 포스트가 없는 상태, **When** 카테고리 페이지를 조회, **Then** 인기 블로그 통계 섹션이 비어있음을 나타내는 메시지가 표시된다

---

### User Story 3 - 카테고리 상세 페이지 탐색 (Priority: P1)

독자가 카테고리 통계 Grid의 특정 항목을 클릭하면 해당 카테고리에 속한 모든 포스트 목록을 볼 수 있다. 이 상세 페이지는 홈페이지의 포스트 상세 페이지와 동일한 레이아웃을 따른다.

**Why this priority**: 카테고리별로 포스트를 필터링하여 탐색하는 것은 사용자의 핵심 요구사항이며, 통계 페이지의 최종 목적이다. 이 기능 없이는 카테고리 통계가 단순 정보 제공에만 그친다.

**Independent Test**: 특정 카테고리를 클릭했을 때 해당 카테고리의 포스트만 표시되고, 페이지 레이아웃이 홈페이지 포스트 상세 페이지와 일치하는지 확인할 수 있다.

**Acceptance Scenarios**:

1. **Given** 카테고리 통계 페이지에 접속한 상태, **When** 특정 카테고리 Grid 항목을 클릭, **Then** 해당 카테고리의 포스트 목록 페이지로 이동한다
2. **Given** 카테고리 상세 페이지에 접속한 상태, **When** 페이지가 로드됨, **Then** 해당 카테고리에 속한 포스트만 표시된다
3. **Given** 카테고리 상세 페이지에 접속한 상태, **When** 페이지 레이아웃을 확인, **Then** 홈페이지의 포스트 상세 페이지와 동일한 구조(레이아웃, 스타일, 컴포넌트)를 사용한다
4. **Given** 특정 카테고리를 선택한 상태, **When** 포스트 항목을 클릭, **Then** 해당 포스트의 전체 내용 페이지로 이동한다
5. **Given** 카테고리에 포스트가 없는 상태, **When** 해당 카테고리를 클릭, **Then** "이 카테고리에 포스트가 없습니다" 메시지가 표시된다

---

### Edge Cases

- **빈 카테고리**: 포스트가 하나도 없는 카테고리를 클릭했을 때 적절한 안내 메시지가 표시되어야 한다
- **대량의 카테고리**: 카테고리가 50개 이상일 때 페이지 성능과 가독성이 유지되어야 한다
- **멀티 태그 포스트**: 하나의 포스트가 여러 태그를 가진 경우, 각 태그의 포스트 수에 중복 카운트되어야 한다
- **태그 삭제**: 모든 포스트에서 제거된 태그는 카테고리 목록에 표시되지 않아야 한다
- **한글/영문 혼합 태그**: 한글과 영문이 혼합된 태그명이 정렬 시 올바르게 처리되어야 한다
- **동시 접근**: 여러 사용자가 동시에 카테고리 통계를 조회할 때 성능 저하가 없어야 한다

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 시스템은 블로그 상단 네비게이션에 '카테고리' 링크를 제공해야 한다
- **FR-002**: 시스템은 카테고리 통계 페이지에서 모든 태그 목록과 각 태그의 포스트 수를 표시해야 한다
- **FR-003**: 시스템은 카테고리 통계 페이지를 인기 블로그 통계 Grid와 카테고리별 블로그 통계 Grid 두 섹션으로 구성해야 한다
- **FR-004**: 시스템은 인기 블로그 통계 Grid에 is_popular 플래그가 true인 포스트들을 표시해야 한다
- **FR-005**: 시스템은 카테고리별 블로그 통계 Grid를 정렬된 순서로 표시해야 한다 (기본: 가나다/알파벳 순)
- **FR-006**: 시스템은 각 카테고리 Grid 항목 클릭 시 해당 카테고리의 포스트 목록 페이지로 이동해야 한다
- **FR-007**: 시스템은 카테고리 상세 페이지에서 선택된 카테고리에 속한 포스트만 필터링하여 표시해야 한다
- **FR-008**: 시스템은 카테고리 상세 페이지의 레이아웃과 구조를 홈페이지의 포스트 목록 페이지와 동일하게 유지해야 한다
- **FR-009**: 시스템은 포스트가 없는 카테고리를 클릭했을 때 적절한 안내 메시지를 표시해야 한다
- **FR-010**: 시스템은 각 카테고리의 포스트 수를 정확하게 카운트하여 표시해야 한다
- **FR-011**: 시스템은 카테고리 통계 페이지가 모바일 및 데스크톱에서 반응형으로 표시되도록 해야 한다
- **FR-012**: 시스템은 카테고리 상세 페이지에서 포스트를 발행일 기준 최신순으로 정렬해야 한다

### Key Entities

- **Category/Tag**: 블로그 포스트를 분류하는 태그. 속성: 태그 이름, 슬러그, 포스트 수(계산됨)
- **Post**: 블로그 포스트. 속성: 제목, 저자, 썸네일, 발행일, is_popular 플래그, 연결된 태그들
- **Category Statistics**: 각 태그의 집계 데이터. 속성: 태그 이름, 해당 태그를 가진 포스트 총 수

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 사용자가 카테고리 링크를 클릭한 후 2초 이내에 카테고리 통계 페이지가 로드되어야 한다
- **SC-002**: 사용자가 특정 카테고리를 클릭하면 3초 이내에 해당 카테고리의 포스트 목록이 표시되어야 한다
- **SC-003**: 카테고리 통계 페이지는 100개 이상의 카테고리가 있을 때도 5초 이내에 로드되어야 한다
- **SC-004**: 사용자가 카테고리별로 포스트를 탐색하는 작업을 3클릭 이내에 완료할 수 있어야 한다 (홈 → 카테고리 페이지 → 특정 카테고리 → 포스트)
- **SC-005**: 카테고리 상세 페이지의 레이아웃이 홈페이지 포스트 목록과 시각적으로 일관성 있게 유지되어야 한다 (사용자 테스트에서 90% 이상이 동일한 경험으로 인식)
- **SC-006**: 각 카테고리의 포스트 수가 실제 데이터베이스의 포스트 수와 100% 일치해야 한다
- **SC-007**: 모바일 기기에서 카테고리 Grid가 가독성 있게 표시되어야 한다 (최소 2열 그리드 레이아웃)
